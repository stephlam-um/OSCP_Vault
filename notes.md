# 0. 評審視角下的修訂重點

原方案的主題有澳門在地性，也有阿里雲與 QwenPaw 的技術連接；但若以競賽評審角度看，還需要從「願景型平台」收斂成「現場可演示、結果可驗證、Agent 能力可看見」的原型。修訂後的方案重點如下：

|                                  |                                                                                            |
| -------------------------------- | ------------------------------------------------------------------------------------------ |
| **原方案風險**                        | **修訂策略**                                                                                   |
| 「建立澳門首個文化知識庫」等表述過大，短期難以證明。       | 改為「以 3–5 間老字號為 MVP 樣本，展示從訪談到資產包的閉環」。                                                       |
| Agent 分工有名稱，但多步執行、工具調用、失敗恢復不夠明確。 | 加入 Paw-Interviewer、Paw-Archivist、Paw-Verifier、Paw-Graph、Paw-Navigator、Paw-Copilot 的可觀察任務鏈。 |
| 商業價值偏宏觀，容易像政府平台計劃書。              | 加入 B 端「商戶上架包」、C 端「記憶尋寶路線」、G 端「文化資產風險看板」。                                                   |
| 缺少評測方式，評審難判斷完成度。                 | 加入抽取準確率、證據覆蓋率、人工審核通過率、多語種、演示成功率等指標。                                                        |
| 文化保育容易變成 AI 自動編故事。               | 加入「證據錨點、信心分級、商戶審核、不可公開欄位、刪除權」等治理設計。                                                        |
